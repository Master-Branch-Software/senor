# Sources — 04 Blogs and Articles

## Primary citations

- Nielsen Norman Group, "The Inverted Pyramid: Writing for Comprehension." <https://www.nngroup.com/articles/inverted-pyramid/>. Source for inverted pyramid as the structural response to web scanning behavior.
- Wikipedia, "Inverted pyramid (journalism)." <https://en.wikipedia.org/wiki/Inverted_pyramid_(journalism)>. Source for the lede/nut graf/body/background/tail structure.
- Kathy Widenhouse, "How to BLUF When You're Writing Articles." <https://medium.com/the-book-mechanic/how-to-bluf-when-youre-writing-articles-797584223827>. Source for BLUF style in article writing.
- Daniele Procida et al., "Diátaxis." <https://diataxis.fr/>. Cross-reference for the four documentation types — relevant where blog posts function as how-to or explanation; full coverage in `technical-writing.md`.
- AP Stylebook (Associated Press, 2024). House style for journalism; canonical reference for news writing conventions, attribution patterns, and dateline usage.

## Secondary references

- Ohio State Pressbooks, "Inverted pyramid style." <https://ohiostate.pressbooks.pub/stratcommwriting/chapter/inverted-pyramid-style/>.
- Crazy Egg, "The Blog Post Structure That Google (and Readers) Love." <https://www.crazyegg.com/blog/blog-post-structure/>.
- Brafton, "How to Structure a Blog Post." <https://www.brafton.com/blog/content-writing/blog-post-structure/>.
- Roy Peter Clark, _Writing Tools: 50 Essential Strategies for Every Writer_ (Little, Brown, 2006). Particularly Tools 1–5 on subject-verb-object order, sentence rhythm, and active voice.

## Curated tool catalog

- CoSchedule Headline Studio — scores headlines against length, balance, and emotional language; useful as a sanity check, not a target.
- Hemingway Editor — reading-grade and complexity check on long-form prose; flag adverbs, passive voice, complex sentences.
- Grammarly / LanguageTool — grammar pass; configure custom dictionaries for technical terms to suppress false positives.

## Notes for editors

The genre table is intentionally opinionated: typical lengths and POVs reflect publishing convention, not hard rules. When extending, prefer citing one canonical published example per genre over abstract description.

The "endings to cut" list overlaps with `editing-and-anti-patterns.md` § Common AI-default endings; cross-reference rather than restate.
