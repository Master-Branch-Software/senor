The problem

AI outputs the median. The median is bad.

Every model trains on the same corpus: Tailwind tutorials, dribbble shots, SaaS landing-page boilerplate. Ask for a website and you get an indigo gradient, three feature cards, and copy that says "empower your workflow." Ask for code and you get div soup — no semantic structure, missing ARIA attributes, hard-coded pixel values everywhere.

It's not a hallucination problem. It's a training-data problem. The median web is mediocre, and AI is very good at producing the median.

These guidelines are built from engineering blogs, official specifications, and professional codebases. The rules are concrete enough to enforce: specific color space, specific HTML elements, specific copy patterns. What a senior engineer follows without needing to think about it.

The web design module ships first. Backend architecture, TypeScript patterns, Python, and more are in development.

Setup

Set up in minutes.

1.

Clone the repository

Get SKILL.md and all 15 reference chapters from GitHub. Plain Markdown files. Nothing to install.

git clone github.com/Master-Branch-Software/ai-web-designer
2.

Add to Claude Code

Copy SKILL.md into your project's .claude/ folder, or add it as a global skill in Claude Code settings. It loads automatically on every task.

cp web-design/SKILL.md .claude/web-design.md
3.

Start building

Claude reads the guidelines on every web task. No prompt engineering. No reminders. It just follows the rules and stops producing the defaults.
