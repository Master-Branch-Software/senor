// the problem

AI produces the median. The median is bad.

Every model trains on the same data: Tailwind tutorials, dribbble shots, SaaS boilerplate. The output is predictable — indigo gradients, div soup, copy full of "empower" and "streamline." It's not wrong exactly. It's just the average of everything that's already out there.

design

Visual decisions that don't look generated

OKLCH-based color palettes instead of copied Tailwind values. Type scales with a modular ratio. Layout archetypes chosen for the brief, not picked from the training data. Dark mode built as a parallel system, not a CSS inversion.

  • OKLCH palettes
  • modular type scale
  • dark mode
  • layout archetypes

/* Not this */

--color: #6366f1; /* indigo-500 */

 

/* This — OKLCH 12-step scale */

--c-accent: oklch(0.48 0.13 55);

--c-accent-hi: oklch(0.74 0.10 55);

--c-step-5: clamp(3.05rem, 2.75rem + 1.5vw, 4.74rem);

code quality

Semantic HTML that the accessibility tree can read

AI-generated code has a recognizable fingerprint: <div onClick> instead of <button>, missing aria-expanded on disclosure widgets, unlabelled SVG icons, landmarks that are just styled <div>s. The guidelines make every one of these a specific rule.

  • WCAG 2.2 AA
  • ARIA patterns
  • keyboard nav
  • semantic HTML

<!-- Generated default -->

<div class="btn" onClick="…">

  Submit</div>

 

<!-- With guidelines -->

<button type="submit"

  aria-label="Save changes">

  Save changes</button>

copy

Copy without the 24 AI fingerprints

Stylometric research identifies LLM-generated text with >90% accuracy from structural patterns: -ing participial stacking, nominalization chains, contractions at 0.00 per chunk. The guidelines document all 24 patterns and give specific edits for each.

Clone. Copy. Build.

step 01

Clone the repository

Plain Markdown files. No install, no build step, no dependencies.

git clone github.com/Master-Branch-Software/ai-web-designer

step 02

Add to Claude Code

Copy SKILL.md to your project's .claude/ folder. It loads automatically on every task.

cp web-design/SKILL.md .claude/web-design.md

step 03

Ask Claude to build

Claude reads the guidelines on every web task. No prompt engineering. No reminders. It just follows the rules.
